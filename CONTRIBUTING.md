# Contributing to ChatBridge

First off, thank you for considering contributing to ChatBridge! It's people like you that make ChatBridge such a great tool for the AI community.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct:
- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and experiences.
- Gracefully accept constructive criticism.
- Focus on what is best for the community.
- Show empathy towards other community members.

## How Can I Contribute?

### Reporting Bugs
- **Check for existing issues**: Before opening a new issue, please search the issue tracker to see if the bug has already been reported.
- **Be descriptive**: When reporting a bug, include as much detail as possible. Steps to reproduce, expected behavior, and actual behavior are essential.
- **Screenshots**: If applicable, attach screenshots or GIFs to help illustrate the problem.

### Suggesting Enhancements
- **Check for existing suggestions**: Like with bugs, search the issues first.
- **Explain the "Why"**: Describe why this enhancement would be useful to most ChatBridge users.

### Pull Requests
1.  **Fork the repo** and create your branch from `main`.
2.  **Follow the style**: Keep your code consistent with the existing codebase (Vanilla JavaScript, clean CSS).
3.  **Test your changes**: Ensure the extension still works across multiple LLM platforms.
4.  **Document your changes**: Update the README if you add new features or supported platforms.
5.  **Submit the PR**: Provide a clear description of what you've changed and why.

## Supported Platforms (Selectors)
If you're adding support for a new platform, please update `content/extractor.js` with the appropriate CSS selectors. Ensure you handle both user and assistant roles correctly.
